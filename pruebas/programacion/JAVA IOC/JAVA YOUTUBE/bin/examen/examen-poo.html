Ejercicio 0.1 — Clase Libro con atributos y constructor

Crea una clase Libro en Java con los siguientes requisitos:

  • Atributos privados: titulo (String), autor (String), precio (double).
  • Constructor que reciba los 3 parámetros y use this para distinguir atributos de parámetros.
  • Getters para los 3 atributos.
  • Setter para precio solamente (con validación: no permitir precios negativos).
  • Método mostrarDatos() que imprima toda la información del libro.

En el main(), crea dos libros, asígnales sus datos y muéstralos.

✅ Solución 0.1

class Libro {
    private String titulo;
    private String autor;
    private double precio;

    // Constructor con this para diferenciar atributos de parametros
    Libro(String titulo, String autor, double precio) {
        this.titulo = titulo;
        this.autor = autor;
        this.precio = precio;
    }

    // Getters
    String getTitulo() { return titulo; }
    String getAutor() { return autor; }
    double getPrecio() { return precio; }

    // Setter solo para precio (con validacion)
    void setPrecio(double precio) {
        if (precio >= 0) {
            this.precio = precio;
        } else {
            System.out.println("Error: el precio no puede ser negativo");
        }
    }

    void mostrarDatos() {
        System.out.println("Titulo: " + titulo);
        System.out.println("Autor: " + autor);
        System.out.println("Precio: " + precio + " euros");
        System.out.println("---");
    }

    public static void main(String[] args) {
        Libro libro1 = new Libro("Cien anos de soledad",
            "Gabriel Garcia Marquez", 19.95);
        Libro libro2 = new Libro("1984", "George Orwell", 14.50);

        libro1.mostrarDatos();
        libro2.mostrarDatos();
    }
}

  • Dos constructores usando this() para reutilizar:

    • Estudiante(String nombre, int edad) — llama al primero con curso = "Sin asignar".

Ejercicio 1.1 — Identificar elementos POO

Dado el siguiente escenario: "Tenemos una aplicación para gestionar una biblioteca. Necesitamos representar libros, cada uno con su título, autor, ISBN, año de publicación, y si está prestado o no. Los libros se pueden prestar, devolver y consultar."

Identifica y explica en el contexto de este escenario:

  1. ¿Cuál sería la clase?
  2. ¿Qué son los atributos?
  3. ¿Qué son los métodos?
  4. ¿Qué sería una instancia? Pon un ejemplo concreto.
  5. ¿Qué es el estado de un objeto? Pon un ejemplo.
  6. ¿Qué es un mensaje entre objetos? Pon un ejemplo.

✅ Solución 1.1

  1. Clase: Libro — el molde o plantilla para crear objetos libro.
  2. Atributos: titulo, autor, isbn, anioPublicacion, prestado (tipo boolean). Definen el estado de cada libro.
  3. Métodos: prestar(), devolver(), consultarDatos(). Definen el comportamiento del libro.
  4. Instancia: Un libro concreto creado a partir de la clase. Ej: Libro miLibro = new Libro("Cien Años de Soledad", "García Márquez", "978-84-376-0494-7", 1967);
  5. Estado: El valor actual de todos los atributos de un objeto. Ej: para miLibro el estado podría ser {titulo="Cien Años de Soledad", autor="García Márquez", isbn="978-84-376-0494-7", anioPublicacion=1967, prestado=true}

Ejercicio 1.2 — Crisis del software y convenciones

Responde brevemente:

  1. ¿Qué problemas surgieron en la llamada "crisis del software" de los años 60-70 que llevaron al desarrollo de la POO?
  2. Explica las convenciones de nomenclatura en Java para: clases, variables y métodos.
  3. ¿Por qué decimos que una clase es un "molde" o "plantilla"? ¿Qué relación tiene con el concepto de "instancia"?

✅ Solución 1.2

  1. Crisis del software: Los programas crecían y el paradigma orientado a procedimientos generaba: código difícil de mantener, código poco reutilizable (código duplicado), proyectos fuera de plazo y presupuesto, y software de baja calidad. La POO surgió como solución.
  2. Convenciones Java:
    • Clases: PascalCase (primera letra mayúscula). Ej: Persona, CuentaBancaria, Coche.
    • Variables: camelCase (primera minúscula, siguientes mayúsculas). Ej: nombrePersona, saldoCuenta.
    • Métodos: camelCase + paréntesis. Ej: obtenerDatos(), cambiarProfesion().
  3. Clase como molde: Una clase define la estructura (atributos) y el comportamiento (métodos) que tendrán todos los objetos de ese tipo. Una instancia es un objeto concreto creado a partir de ese molde, con valores específicos para sus atributos. Así como un molde de galletas produce muchas galletas, una clase produce muchas instancias, cada una con su propio estado.

Ejercicio 2.1 — Diseñar una clase Coche

Diseña una clase Coche en Java con:

  • Atributos: marca (String), modelo (String), velocidad (int, velocidad actual en km/h).
  • Un constructor con 3 parámetros que inicialice todos los atributos.
  • Un método de instancia acelerar(int incremento) que aumente la velocidad.
  • Un método de instancia frenar(int decremento) que disminuya la velocidad.
  • Un método mostrarDatos() que imprima los datos del coche.

Escribe el código Java de la clase completa.

✅ Solución 2.1

class Coche {
    // Atributos
    String marca;
    String modelo;
    int velocidad;

    // Constructor
    Coche(String marca, String modelo, int velocidad) {
        this.marca = marca;
        this.modelo = modelo;
        this.velocidad = velocidad;
    }

    // Métodos de instancia
    void acelerar(int incremento) {
        velocidad += incremento;
    }

    void frenar(int decremento) {
        velocidad -= decremento;
        if (velocidad < 0) velocidad = 0;
    }

    void mostrarDatos() {
        System.out.println("Marca: " + marca);
        System.out.println("Modelo: " + modelo);
        System.out.println("Velocidad: " + velocidad + " km/h");
    }
}

Ejercicio 3.1 — Clase CuentaBancaria con encapsulación

Crea una clase CuentaBancaria con:

  • Atributos privados: id (int), saldo (double).
  • Constructor que reciba el id e inicialice saldo a 0.
  • Métodos públicos:
    • ingresar(double cantidad) — solo si cantidad > 0.
    • retirar(double cantidad) — solo si hay saldo suficiente y cantidad > 0.
    • verSaldo() — muestra el saldo.

Escribe la clase completa y un main() que cree una cuenta, ingrese 500€, retire 200€ y muestre el saldo final.

✅ Solución 3.1

class CuentaBancaria {
    private int id;
    private double saldo;

    CuentaBancaria(int id) {
        this.id = id;
        this.saldo = 0;
    }

    void ingresar(double cantidad) {
        if (cantidad > 0) {
            saldo += cantidad;
            System.out.println("Ingresados " + cantidad + " €");
        } else {
            System.out.println("ERROR: Cantidad no válida");
        }
    }

    void retirar(double cantidad) {
        if (cantidad <= 0) {
            System.out.println("ERROR: Cantidad no válida");
        } else if (saldo >= cantidad) {
            saldo -= cantidad;
            System.out.println("Retirados " + cantidad + " €");
        } else {
            System.out.println("ERROR: Saldo insuficiente");
        }
    }

    void verSaldo() {
        System.out.println("Saldo de cuenta " + id + ": " + saldo + " €");
    }

    public static void main(String[] args) {
        CuentaBancaria c = new CuentaBancaria(1);
        c.ingresar(500);
        c.retirar(200);
        c.verSaldo();  // Saldo: 300 €
    }
}

Ejercicio 3.2 — Composición de objetos (Persona + Cuenta)

Partiendo de la clase Persona (con atributos privados nombre, edad) y la clase CuentaBancaria del ejercicio anterior:

  1. Añade a CuentaBancaria un atributo titular de tipo Persona.
  2. Crea un método asignarTitular(Persona p).
  3. En el main(), crea una Persona "María", 30 años, créale una cuenta, asigna a María como titular, ingresa 1000€ y muestra los datos completos (titular + saldo).

¿Qué concepto de la POO estamos aplicando al tener una Persona dentro de CuentaBancaria?

✅ Solución 3.2

class Persona {
    private String nombre;
    private int edad;

    Persona(String nombre, int edad) {
        this.nombre = nombre;
        this.edad = edad;
    }

    String getNombre() { return nombre; }
    int getEdad() { return edad; }

    void mostrarDatos() {
        System.out.println("  Nombre: " + nombre + ", Edad: " + edad);
    }
}

class CuentaBancaria {
    private int id;
    private double saldo;
    private Persona titular;  // COMPOSICIÓN

    CuentaBancaria(int id) {
        this.id = id;
        this.saldo = 0;
    }

    void asignarTitular(Persona p) { this.titular = p; }

    void ingresar(double c) { if (c > 0) { saldo += c; } }

    void mostrarDatos() {
        System.out.println("Cuenta " + id + ":");
        if (titular != null) titular.mostrarDatos();
        System.out.println("  Saldo: " + saldo + " €");
    }

    public static void main(String[] args) {
        Persona maria = new Persona("María", 30);
        CuentaBancaria c = new CuentaBancaria(123);
        c.asignarTitular(maria);
        c.ingresar(1000);
        c.mostrarDatos();
    }
}

Estamos aplicando el concepto de composición: una clase (CuentaBancaria) tiene como atributo otra clase (Persona). Esto permite modelar relaciones del mundo real entre objetos.

V-04: Características de la POO

Abstracción, encapsulamiento, herencia, polimorfismo

Ejercicio 4.1 — Las 4 características de la POO

Explica cada una de las 4 características fundamentales de la POO con tus propias palabras y pon un ejemplo breve de cada una:

  1. Abstracción
  2. Encapsulamiento
  3. Herencia
  4. Polimorfismo

✅ Solución 4.1

  1. Abstracción: Proceso de extraer las características esenciales de un objeto del mundo real, ignorando los detalles irrelevantes. Ej: para una app de una biblioteca, de un libro nos interesa título, autor, ISBN; no el color de la portada o el grosor.
  2. Encapsulamiento: Ocultar la información interna del objeto (atributos private) y permitir el acceso solo a través de métodos públicos (getters/setters). Ej: private int edad con setEdad() que valida que no sea negativa.
  3. Herencia: Mecanismo por el cual una clase (subclase/hija) hereda atributos y métodos de otra (superclase/padre). Ej: Perro extends Animal — Perro hereda todos los atributos y métodos de Animal y puede añadir los suyos propios.
  4. Polimorfismo: Capacidad de un método de comportarse de forma distinta según la clase del objeto que lo invoca. Ej: el método sonido() en Animal — si lo invoca un Perro hace "Guau", si lo invoca un Gato hace "Miau".

Ejercicio 4.2 — Herencia y polimorfismo en acción

Crea una jerarquía de clases donde:

  • Empleado (superclase) con atributos nombre y salario, y método trabajar() que muestre "El empleado [nombre] está trabajando".
  • Programador (subclase) que hereda de Empleado, añade el atributo lenguaje y sobreescribe trabajar() para que muestre "El programador [nombre] está escribiendo código en [lenguaje]".
  • Disenador (subclase) que hereda de Empleado, añade herramienta y sobreescribe trabajar() para mostrar "El diseñador [nombre] está diseñando con [herramienta]".

Crea un main() que demuestre el polimorfismo.

✅ Solución 4.2

class Empleado {
    protected String nombre;
    protected double salario;

    Empleado(String nombre, double salario) {
        this.nombre = nombre;
        this.salario = salario;
    }

    void trabajar() {
        System.out.println("El empleado " + nombre + " está trabajando");
    }
}

class Programador extends Empleado {
    private String lenguaje;

    Programador(String nombre, double salario, String lenguaje) {
        super(nombre, salario);
        this.lenguaje = lenguaje;
    }

    @Override
    void trabajar() {
        System.out.println("El programador " + nombre
            + " está escribiendo código en " + lenguaje);
    }
}

class Disenador extends Empleado {
    private String herramienta;

    Disenador(String nombre, double salario, String herramienta) {
        super(nombre, salario);
        this.herramienta = herramienta;
    }

    @Override
    void trabajar() {
        System.out.println("El diseñador " + nombre
            + " está diseñando con " + herramienta);
    }
}

// Demostración de polimorfismo:
public static void main(String[] args) {

    e2.trabajar();  // El diseñador Luis está diseñando con Figma
}

V-05: Diseño orientado a objetos en UML

Diagrama de clases UML, relaciones: asociación, agregación, composición, cardinalidad

Ejercicio 5.1 — Leer y explicar un diagrama UML

Dado el siguiente esquema de relaciones UML:

  Pedido ── 1 ──> Cliente
  Pedido <>── * ──> LineaPedido
  Producto ── 1..* ──> Categoria

Responde:

  1. ¿Qué tipo de relación hay entre Pedido y Cliente? Explica la cardinalidad.
  2. ¿Qué tipo de relación hay entre Pedido y LineaPedido? ¿Por qué se usa composición (rombo negro)?
  3. ¿Qué significa la cardinalidad 1..* entre Producto y Categoria?
  4. ¿Qué es la navegabilidad en UML y cómo se representa?

✅ Solución 5.1

  1. Asociación simple entre Pedido y Cliente. Cardinalidad: un pedido pertenece a 1 cliente, y un cliente puede tener * (muchos) pedidos. La flecha indica navegabilidad de Pedido → Cliente.
  2. Composición (rombo negro en Pedido). Las líneas de pedido son "parte de" un pedido. Si el pedido se elimina, sus líneas también se eliminan. Una línea de pedido no puede existir sin su pedido.
  3. Un producto pertenece a 1 o más categorías (1..*), y una categoría puede tener * productos. Es una asociación muchos-a-muchos.
  4. Navegabilidad: indica quién conoce a quién. Se representa con una flecha en el extremo del destino. Si la flecha va de Pedido → Cliente, significa que Pedido tiene una referencia a Cliente, pero no al revés. Sin flecha = bidireccional.

Ejercicio 5.2 — Diseñar UML y código para Universidad

Modela con UML textual y escribe el código Java para un sistema universitario donde:

  • Un Profesor tiene nombre, email y puede impartir varios Cursos (un curso solo lo imparte un profesor).
  • Un Curso tiene nombre, código y varios Estudiantes.
  • Un Estudiante tiene nombre, número de matrícula y puede estar en varios cursos.

Identifica los tipos de relación y las cardinalidades.

✅ Solución 5.2

Diagrama UML textual:

  Profesor ── 1 ──> * Curso
  Curso * ──> * Estudiante
  • Profesor → Curso: Asociación 1 → *. Un profesor imparte varios cursos; un curso tiene un solo profesor.
  • Curso ↔ Estudiante: Asociación * → * (bidireccional). Un curso tiene muchos estudiantes; un estudiante puede estar en muchos cursos.

class Curso {

    void agregarEstudiante(Estudiante e) { estudiantes.add(e); }
}

class Estudiante {
    private String nombre;
    private int numMatricula;
    private ArrayList<Curso> cursos = new ArrayList<>();

    Estudiante(String n, int m) { nombre = n; numMatricula = m; }
    void matricularEn(Curso c) { cursos.add(c); c.agregarEstudiante(this); }
}

Ejercicio 6.1 — Validar mapa de objetos (Biblioteca)

Tenemos las siguientes clases con sus cardinalidades UML:

  Biblioteca ── * ──> Libro
  Libro ── 1 ──> Autor

Determina si estos mapas de objetos son VÁLIDOS o INVÁLIDOS y explica por qué:

  1. BibliotecaA → Libro1, Libro2 (AutorA, AutorB) | BibliotecaB → (sin libros) | BibliotecaC → Libro3 (AutorA)
  2. BibliotecaA → Libro1, Libro2 | Libro1 → AutorA, AutorB (un libro con dos autores)

✅ Solución 6.1

Ejercicio 10.2 — Saturación vs Circular

Explica la diferencia entre las dos estrategias de manejo de límites. Después, modifica la clase Televisor del ejercicio anterior para que el canal use estrategia circular:

  • Si canal = 99 y subimos → canal = 1
  • Si canal = 1 y bajamos → canal = 99

¿Qué ventajas tiene cada estrategia? ¿En qué contexto usarías cada una?

✅ Solución 10.2

Diferencias:

  • Saturación: Al llegar al límite, el valor se queda ahí. No puede pasar del máximo ni del mínimo. Ej: volumen 0-15, si estás en 15 y subes, sigue en 15.
  • Circular: Al llegar al límite, "da la vuelta". Ej: canales 1-99, si estás en 99 y subes, vuelve a 1.

Setter circular:

void setCanal(int valor) {
    if (valor > 99) {
        this.canal = 1;
    } else if (valor < 1) {
        this.canal = 99;
    } else {
        this.canal = valor;
    }
}

¿Cuándo usar cada una?

  • Saturación: Para características que tienen un mínimo y máximo absolutos que no tiene sentido "dar la vuelta" (volumen, temperatura, brillo).
  • Circular: Para valores que forman un ciclo (canales de TV, días de la semana, meses del año, opciones de menú).

false  // c es new String() -> nuevo objeto en heap (referencia distinta)
true   // equals() compara CONTENIDO, no referencia
true   // El compilador concatena "Hola " + "Mundo" en tiempo de compilación
       // generando "Hola Mundo", que ya está en el pool

✅ Solución 20.1

Shallow copy: Copia solo las referencias del array. El nuevo array apunta a los mismos objetos. Si modificamos un objeto desde el array original, también se modifica en el copiado (y viceversa). Se hace con clone() o Arrays.copyOf().

Deep copy: Crea un nuevo array y nuevos objetos para cada posición. Los arrays original y copiado son completamente independientes.

¿Cuándo usar cada una?

  • Shallow copy suficiente: Cuando los elementos del array son inmutables (no tienen setters), como String, Integer, o clases inmutables propias.

¿Por qué reutilizar setters en el constructor?

class Empleado {

    void setPermanencia(int p) {
        if (p < 0) {

    Empleado(String nombre, int permanencia, double salario) {
        this.nombre = nombre;
        setPermanencia(permanencia);  // Usa la validación del setter
        // ...
    }

    public static void main(String[] args) {
        // Con setter en constructor:
        Empleado e = new Empleado("Test", -3, 1000);

Ejercicio 23.1 — Puntos distintos y varargs

Crea una clase Punto (con x, y) e implementa:

  1. Un método estático distintos(Punto p1, Punto p2) que devuelva true si están en distinta posición.
  2. Un método estático distintos(Punto... puntos) con varargs que devuelva true si todos los puntos son distintos entre sí.
  3. Un método estático distancia(Punto p1, Punto p2) que devuelva la distancia.

✅ Solución 23.1

class Punto {
    private int x, y;
    Punto(int x, int y) { this.x = x; this.y = y; }
    int getX() { return x; }
    int getY() { return y; }
    public String toString() { return "(" + x + "," + y + ")"; }

    // Dos puntos distintos
    static boolean distintos(Punto p1, Punto p2) {
        return p1.x != p2.x || p1.y != p2.y;
    }

    // N puntos distintos (varargs)
    static boolean distintos(Punto... puntos) {
        for (int i = 0; i < puntos.length; i++) {
            for (int j = i + 1; j < puntos.length; j++) {
                if (puntos[i].x == puntos[j].x && puntos[i].y == puntos[j].y) {
                    return false;  // Dos iguales encontrados
                }
            }
        }
        return true;
    }

    // Distancia euclídea
    static double distancia(Punto p1, Punto p2) {
        return Math.sqrt(Math.pow(p1.x - p2.x, 2) + Math.pow(p1.y - p2.y, 2));
    }

    public static void main(String[] args) {
        Punto a = new Punto(0, 0);
        Punto b = new Punto(1, 0);
        Punto c = new Punto(0, 0);  // Igual que a

        System.out.println("a != b: " + distintos(a, b));  // true
        System.out.println("a != c: " + distintos(a, c));  // false
        System.out.println("a,b distintos: " + distintos(a, b)); // true
        System.out.println("a,b,c distintos: " + distintos(a, b, c)); // false
        System.out.println("Distancia (0,0)-(1,0): " + distancia(a, b)); // 1.0
    }
}

Ejercicio 23.2 — Área del triángulo con Herón

Usando la clase Punto del ejercicio anterior, implementa un método:

static void areaTriangulo(Punto p1, Punto p2, Punto p3)

Que calcule y muestre:

  • El tipo de triángulo: Equilátero, Isósceles o Escaleno.
  • El área usando la fórmula de Herón.
  • Si los puntos no forman un triángulo (área = 0), debe mostrar un error.

Fórmula de Herón: s = (a + b + c) / 2; Área = √(s(s-a)(s-b)(s-c))

Prueba con: (0,0), (1,0), (0,1) → isósceles, área 0.5 y (0,0), (1,1), (2,2) → error.

✅ Solución 23.2

class TrianguloUtils {
    // Determina tipo de triángulo por sus lados
    static String obtenerTipo(double a, double b, double c) {
        if (a == b && b == c) return "Equilátero";
        if (a != b && b != c && a != c) return "Escaleno";
        return "Isósceles";
    }

    static void areaTriangulo(Punto p1, Punto p2, Punto p3) {
        // Verificar que los 3 puntos sean distintos
        if (!Punto.distintos(p1, p2, p3)) {
            System.out.println("Error: puntos repetidos");
            return;
        }

        // Calcular lados
        double a = Punto.distancia(p1, p2);
        double b = Punto.distancia(p2, p3);
        double c = Punto.distancia(p3, p1);

        // Semiperímetro
        double s = (a + b + c) / 2;

        // Herón
        double area = Math.sqrt(s * (s - a) * (s - b) * (s - c));

        if (area == 0) {
            System.out.println("Error: no forman triángulo (colineales)");
            return;
        }

        String tipo = obtenerTipo(a, b, c);
        System.out.println("Triángulo " + tipo + " | Área: "
            + String.format("%.2f", area));
    }

    public static void main(String[] args) {
        Punto p1 = new Punto(0, 0);
        Punto p2 = new Punto(1, 0);
        Punto p3 = new Punto(0, 1);

        areaTriangulo(p1, p2, p3);  // Isósceles, 0.50

        Punto p4 = new Punto(2, 2);
        areaTriangulo(p1, p2, p4);  // Error
    }
}

class Punto { int x, y; String nombre;

class GraficoLineas2D {
    private ArrayList puntos;

    GraficoLineas2D() { this.puntos = new ArrayList<>(); }
    GraficoLineas2D(ArrayList l) { this(); setPuntos(l); }

    private boolean esPositivo(Punto p) { return p.getX() >= 0 && p.getY() >= 0; }

    void agregarPunto(Punto p) {
        if (puntos.isEmpty()) { puntos.add(p); return; }
        Punto ult = puntos.get(puntos.size() - 1);
        if (esPositivo(p) && !(p.getX() == ult.getX() && p.getY() == ult.getY())
            && p.getX() >= ult.getX()) {
            puntos.add(p);
        } else {
            System.out.println("ERROR: punto " + p + " inválido");
        }
    }

    void eliminarPunto() {
        if (!puntos.isEmpty()) puntos.remove(puntos.size() - 1);
    }

    void setPuntos(ArrayList l) {
        if (validarPuntos(l)) this.puntos = new ArrayList<>(l);
        else System.out.println("ERROR: lista inválida");
    }

    private boolean validarPuntos(ArrayList l) {
        if (l.isEmpty()) return true;
        if (!esPositivo(l.get(0))) return false;
        for (int i = 1; i < l.size(); i++) {
            Punto act = l.get(i), ant = l.get(i - 1);
            if (!esPositivo(act) || (act.getX() == ant.getX() && act.getY() == ant.getY())
                || act.getX() < ant.getX()) return false;
        }
        return true;
    }

    void mostrarGrafico() {
        for (Punto p : puntos) System.out.println("  " + p);
    }
}

import java.util.ArrayList;

class Cliente {

    private LocalDate fechaNacimiento;

    Cliente(String n, String a, String d, String l, LocalDate fn) {

    private double saldo;

Ejercicio 27.1 — Clase Cita con redondeo de minutos

Implementa una clase Cita con:

  • Atributos: hora (0-23), minutos (redondeado a múltiplo de 5), titulo, texto.
  • Setter setMinutos(int m) que redondee:
    • Si el resto de dividir entre 5 es ≤ 2 → redondea hacia abajo.
    • Si el resto es > 2 → redondea hacia arriba.
    • Ej: 14→15, 21→20, 59→0 (y suma 1 a hora si hora < 23).
  • Setter setHora(int h) que valide 0-23 (si no, pone 0).
  • Método toString() con formato "(HH:mm) [Título] texto".

✅ Solución 27.1

class Cita {
    private int hora, minutos;
    private String titulo, texto;

    Cita(int hora, int minutos, String titulo, String texto) {
        setHora(hora);
        setMinutos(minutos);
        this.titulo = titulo;
        this.texto = texto;
    }

    void setHora(int h) { this.hora = (h >= 0 && h <= 23) ? h : 0; }
    int getHora() { return hora; }
    int getMinutos() { return minutos; }

    void setMinutos(int m) {
        if (m < 0 || m >= 60) { this.minutos = 0; return; }
        int mod = m % 5;
        if (mod == 0) this.minutos = m;
        else if (mod <= 2) this.minutos = m - mod;
        else {
            this.minutos = m + (5 - mod);
            if (this.minutos >= 60) this.minutos = 55;
        }
    }

    public String toString() {
        return String.format("(%02d:%02d) [%s] %s", hora, minutos, titulo, texto);
    }

    public static void main(String[] args) {
        Cita c = new Cita(11, 21, "Reunión", "Java");
        System.out.println(c);  // (11:20) [Reunión] Java (21→20, resto 1)

        Cita c2 = new Cita(11, 14, "Clase", "POO");
        System.out.println(c2); // (11:15) [Clase] POO (14→15, resto 4)
    }
}

Ejercicio 27.2 — Agenda completa con 365 páginas

Implementa las clases Pagina y Agenda:

  • Pagina tiene día, mes y un ArrayList<Cita>. Métodos: agregarCita() (comprobar que no exista otra en el mismo horario), buscarCita(), borrarCita(), verCitas().
  • Agenda tiene un array fijo de 365 Pagina, un año y una página abierta. El constructor genera las 365 páginas usando un array DIAS_MES con los días de cada mes.
  • Método abrirPagina(int dia, int mes).

En el main(), crea una agenda de 2025, abre la página del 15 de marzo, añade 2 citas y muéstralas.

✅ Solución 27.2

import java.util.ArrayList;

class Pagina {
    private int dia, mes;
    private ArrayList citas = new ArrayList<>();
    Pagina(int d, int m) { dia = d; mes = m; }
    int getDia() { return dia; } int getMes() { return mes; }

    Cita buscarCita(int h, int m) {
        for (Cita c : citas)
            if (c.getHora() == h && c.getMinutos() == m) return c;
        return null;
    }

    boolean agregarCita(int h, int m, String t, String tx) {
        Cita c = new Cita(h, m, t, tx);
        if (buscarCita(c.getHora(), c.getMinutos()) != null) return false;
        citas.add(c); return true;
    }

    void verCitas() {
        String[] meses = {"Enero","Febrero","Marzo","Abril","Mayo","Junio",
                          "Julio","Agosto","Septiembre","Octubre","Noviembre","Diciembre"};
        System.out.println("Citas del " + dia + " de " + meses[mes-1]);
        for (Cita c : citas) System.out.println("  " + c);
    }
}

class Agenda {
    private static final int[] DIAS_MES = {31,28,31,30,31,30,31,31,30,31,30,31};
    private int ano;
    private Pagina[] paginas = new Pagina[365];
    private Pagina abierta;

    Agenda(int ano) {
        this.ano = ano;
        int d = 1, m = 1;
        for (int i = 0; i < paginas.length; i++) {
            paginas[i] = new Pagina(d, m);
            d++;
            if (d > DIAS_MES[m-1]) { d = 1; m++; }
        }
        this.abierta = paginas[0];
    }

    void abrirPagina(int d, int m) {
        for (Pagina p : paginas) {
            if (p.getDia() == d && p.getMes() == m) {
                abierta = p;
                System.out.println("Abierta: " + d + "/" + m + "/" + ano);
                return;
            }
        }
        System.out.println("Fecha no válida");
    }
    Pagina getAbierta() { return abierta; }

    public static void main(String[] args) {
        Agenda a = new Agenda(2025);
        a.abrirPagina(15, 3);
        a.getAbierta().agregarCita(10, 0, "Curso Java", "Herencia y polimorfismo");
        a.getAbierta().agregarCita(12, 30, "Comida", "Con el equipo");
        a.getAbierta().verCitas();
    }
}

Estructura UML: Agenda (1) → Pagina (365, array fijo) → Cita (*, ArrayList). La Agenda compone a las Páginas (composición: si la agenda se destruye, las páginas también). Las Páginas agregan Citas (agregación: una cita podría existir sin página, aunque aquí están fuertemente ligadas).
